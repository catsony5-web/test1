# 소비 분석

카드, 이체, 직접 입력 기록을 사용자의 브라우저 안에만 저장해 월별 지출 흐름과 소비 패턴을 분석하는 정적 웹앱입니다.

GitHub Pages에 올려 친구에게 링크를 공유하면, 각 사용자는 자기 브라우저에서 직접 엑셀을 업로드하거나 수입/지출을 입력해 사용할 수 있습니다. 서버로 카드 내역을 보내지 않고, 저장 데이터는 기본적으로 각 브라우저의 IndexedDB 또는 localStorage에 남습니다.

## 배포 원칙

- 저장소에는 앱 코드와 문서만 포함합니다.
- 개인 엑셀 파일, 카드 내역, 백업 JSON, 테스트 소비 데이터는 커밋하지 않습니다.
- GitHub Pages 배포본은 빈 데이터 상태로 시작합니다.
- 사용자가 업로드한 엑셀과 입력한 거래는 각자 브라우저에만 저장됩니다.
- 백업 JSON은 개인 금융 데이터이므로 저장소가 아니라 개인 보관 위치에 따로 저장해야 합니다.

## GitHub Pages 배포 파일

배포에 필요한 파일은 다음입니다.

```text
index.html
manifest.webmanifest
service-worker.js
app-icon.svg
.nojekyll
README.md
qa/
src/
```

다음 파일은 배포 저장소에 포함하지 않습니다.

```text
*.xlsx, *.xls, *.xlsm
*.csv, *.tsv
*.json
*.ps1, *.cmd, *.bat
backups/, exports/, local-data/, private-data/, sample-data/, test-data/
```

이 제외 규칙은 `.gitignore`에 들어 있습니다.

## GitHub Pages 설정

1. GitHub 저장소에 배포 파일만 커밋합니다.
2. 저장소 Settings > Pages로 이동합니다.
3. Source를 `Deploy from a branch`로 선택합니다.
4. Branch는 `main`, folder는 `/root`를 선택합니다.
5. 배포 주소가 생성되면 친구에게 링크를 공유합니다.

프로젝트 페이지 주소가 `https://사용자명.github.io/저장소명/`처럼 하위 경로여도 작동하도록 앱 파일 경로는 상대 경로를 사용합니다.

## 친구가 사용하는 방법

1. GitHub Pages 링크를 엽니다.
2. `카드/이체 엑셀 불러오기`로 본인 카드사 또는 은행 엑셀 파일을 선택합니다.
3. 미분류 거래는 `미분류 해결` 또는 `소비 달력`, `상세 내역`에서 직접 분류합니다.
4. 수입은 `수입 입력`, 과거 수기 지출은 `과거 거래 입력`, 반복 지출은 `고정 지출`에서 입력합니다.
5. 필요하면 `설정/관리`의 백업 기능으로 개인 JSON 백업 파일을 내려받습니다.

## 개인정보와 저장 방식

- 앱은 정적 HTML/CSS/JS로 동작합니다.
- 별도 서버 데이터베이스가 없습니다.
- 거래, 수입, 고정 지출, 분류 규칙, 설정은 각 사용자 브라우저 저장소에 저장됩니다.
- 같은 링크를 열어도 사용자마다 데이터는 서로 공유되지 않습니다.
- 브라우저 데이터 삭제, 시크릿 모드 종료, 회사 보안 프로그램의 저장소 정리 등으로 기록이 사라질 수 있으므로 백업 기능 사용을 권장합니다.

## 주요 기능

- 카드/이체 엑셀 업로드
- 과거 거래 일괄 입력
- 직접 거래 입력과 정산금 수정
- 소비 달력과 날짜별 거래 확인
- 년도 지출정리
- 월별 섹터 요약과 섹터별 리포트
- 수입 입력과 전체 수입 기록 관리
- 고정 지출 등록과 실제 지출 반영
- 소모품 사용 기록
- 미분류 해결과 분류 규칙 관리
- 선택 백업과 선택 복원

## 개발자가 수정할 때

- 고정 분류 체계: `src/data/categories.js`
- 기본 분류 규칙: `src/data/rules.js`
- 카드/이체 엑셀 필드명 매핑: `src/data/field-aliases.js`
- 보드 섹션 정의: `src/data/board-sections.js`
- 저장소/백업 로직: `src/utils/storage.js`, `src/utils/backup.js`
- 화면별 기능: `src/features/`
- CSS: `src/styles/`

## 배포 전 점검

- `git status --short`에서 개인 엑셀, JSON 백업, 로컬 스크립트가 보이지 않는지 확인합니다.
- `service-worker.js`의 캐시 목록에 개인 파일이 없는지 확인합니다.
- `manifest.webmanifest`가 JSON으로 정상 파싱되는지 확인합니다.
- 브라우저에서 `index.html`을 열어 빈 데이터 상태로 앱이 시작되는지 확인합니다.
